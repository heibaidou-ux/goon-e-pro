# 高岸ERP系统-IoT家庭测试最小设备采购清单（V1.1，2026年5月8日）

**版本**：V1.2
**日期**：2026年5月9日
**文档状态**：草稿
**关联文档**：IOT-02（IoT技术实施与家庭测试方案 V1.0）
**用途**：采购最小设备集合搭建家庭测试环境，验证IoT技术栈（HA + Zigbee + RS485 + MQTT）全链路可靠性。

## 四、软件环境（无需采购）

| 组件 | 部署方式 | 说明 |
|------|---------|------|
| Docker Desktop | Windows/Mac安装 | 或WSL2内Docker CE |
| Home Assistant | Docker容器 `ha:stable` | 核心自动化引擎 |
| Mosquitto MQTT | Docker容器 | MQTT消息总线 |
| Zigbee2MQTT | Docker容器 | Zigbee设备接入 |
| Node-RED（可选） | Docker容器 | 可视化自动化流编排 |

> HA主机用自有电脑（Windows Docker Desktop 或 WSL2），无需单独购置M720q。
>
> **短期测试 vs 长期运行**：Windows Docker Desktop 做11天测试没问题。但正式门店24小时运行时，Windows自动更新/重启会导致HA中断，建议届时迁移至原生Linux（M720q Debian）。测试期间观察WSL2内存占用，战66有48GB RAM完全够用，建议给WSL2分配8GB+。

## 五、网络环境（利旧搭建）

```
互联网 — 光猫 — TP-Link路由器
                    │
                    ├─ VLAN 10（设备/管理）192.168.10.0/24
                    │     ├─ 自有PC（HA主机 Docker）
                    │     ├─ PoE交换机（锐捷/金德店）
                    │     │     ├─ Zigbee网关（PoE供电）
                    │     │     └─ PoE摄像头
                    │     └─ RS485主控制器
                    │
                    └─ VLAN 20（模拟客人WiFi）192.168.20.0/24
                          └─ 客人手机/平板
```

- VLAN划分在TP-Link路由器上完成（需确认路由器型号是否支持802.1Q VLAN）
- PoE交换机用锐捷三层交换机做VLAN trunk + 金德店淘汰交换机做接入
- 暂不涉及Tailscale VPN（门店联网后续再做）

## 六、测试计划（调整后）

| 阶段 | 内容 | 涉及设备 | 验证标准 | 预计耗时 |
|------|------|---------|---------|---------|
| **P1 网络基础** | TP-Link VLAN划分 + PoE交换机配置 | 利旧设备 | VLAN间隔离生效，设备互通 | 1天 |
| **P2 HA核心** | Docker部署HA + Z2M安装 + MQTT配置 | 自有PC、Zigbee网关 | HA Web UI可达，Zigbee网关在线 | 1天 |
| **P3 Zigbee设备** | 门锁配对入网、状态上报验证 | 门锁 | 实体正确上报状态、远程开锁正常 | 1天 |
| **P4 RS485总线** | 灯光面板+继电器模块手拉手接线<br>⚠ 调试方法：先用USB转RS485适配器+战66笔记本直连电机/面板，串口助手发Modbus指令验证。直连能动→问题在串口服务器；直连不动→问题在设备或接线。<br>⚠ 归档要求：用Draw.io或飞书白板记录每个设备的 **Slave ID** 和寄存器地址，格式如 `Room_A_Light: SlaveID=02, HoldingReg=40001` | RS485主控、灯光面板×2、继电器模块 | Modbus TP读写正常、HA调光控制 | 2天 |
| **P5 自动化规则** | 欢迎/退房场景联动 | 门锁+灯光+继电器 | 开门→亮灯、退房→关灯 | 2天 |
| **P6 音视频** | 音箱+蓝牙模块接入 | 音箱×2、蓝牙模块 | 背景音乐播放、蓝牙配对音源切换 | 2天 |
| **P7 安防** | 摄像头接入HA + 录像 | 摄像头 | ONVIF集成、移动侦测 | 1天 |
| **P8 集成联调** | 全链路闭环测试 | 全套设备 | 开门→场景联动→退房→复位 | 1天 |

## 七、验证的全链路闭环

```
模拟客人预约（手动触发）
  → 生成门禁密码
  → 输入密码开门
  → 门锁上报DoorOpen事件到HA（Zigbee→Z2M→MQTT→HA）
  → HA触发Welcome场景：
      灯光面板→开灯（RS485→Modbus TCP）
      继电器→开（预留空调/排风回路，实际空载验证）
      音箱→播放背景音乐（蓝牙/音频输出）
  → 模拟退房（手动触发）
  → HA触发EnergySave场景：
      灯光→关
      继电器→关
      音箱→静音
  → 门锁闭锁
```

**通过标准**：全链路HA自动完成所有设备控制，场景触发到设备动作 ≤ 3秒。

## 八、采购决策确认

| # | 事项 | 结论 |
|---|------|------|
| 1 | TP-Link路由器是否支持VLAN | **已确认**：TL-R6812TP-AC 企业级一体机，支持802.1Q VLAN + SSID绑定VLAN，无需另购软路由 |
| 2 | 门锁选型 | **推荐通通锁 TTL-625**（~200-280元），先买1把验证离线密码机制。备选Aqara N100（更贵但HA原生支持） |
| 3 | 音箱+蓝牙是否纳入 | **纳入**，维持采购。此测试关系到后续是否采购多套K歌音箱的决策 |
| 4 | RS485主控选型 | **推荐有人物联网 USR-TCP232-304**（~80元），高性价比，工业级，Web配置简单。备选微雪Waveshare（~65元） |
| 5 | 采购渠道 | **已确认**：IoT设备（门锁、灯光面板、继电器、Zigbee网关）走淘宝/1688；线材辅材走京东或本地电子市场 |
