## 1. 文档目的

本文档根据需求说明书物联网需求及盈隆店实际布局，编制单一门店的IoT设备采购清单，包括设备类型、数量、选型要求和安装注意事项，供采购招标和安装施工使用。

> **说明**：本文档不指定具体硬件品牌或型号，仅从功能需求角度给出选型约束。具体品牌选型在技术设计阶段确定。本文档已根据盈隆店实际布局（V1.1）修正，移除金德店、盈丰店内容。

## 2. 盈隆店布局说明

盈隆店为90°扇形场地（半径12m，圆心角90°），两条半径墙体交汇形成直角。弧线一侧（约17m）全部为落地窗。房间沿弧线排列，工作间沿右侧半径墙体延伸。

### 2.2 动线说明

1. **入口**：开在右侧半径墙体上
2. **进门正对**：照壁（左半实墙、右半门洞）
3. **穿过门洞即到**：展厅
   - 展厅三面墙围成——面向左半径墙时，**左侧是照壁背面**、**右侧是大会议室外墙**、**正面是左半径墙**
4. **展厅右转**：进入走廊
   - 走廊左手边（沿弧线方向）：大会议室 → 中茶室A → 中茶室B（尽头偏左）→ 大茶室C（尽头偏右，邻工作间）
   - 走廊右手边（沿右半径方向）：工作间
5. **工作间**：沿右侧半径墙体延伸，从入口附近延展至大茶室C，无窗

## 3. 设备清单

### 3.1 整体架构说明

本方案采用三层架构设计，遵循《V6.3硬件需求》的技术路线：

| 层级 | 技术方案 | 设备 |
|------|---------|------|
| **控制层** | Home Assistant 本地运行 | HA主机（联想M720q） |
| **通信层** | Zigbee（无线）+ RS485（有线）+ 以太网 | Zigbee网关、RS485主控、PoE交换机 |
| **设备层** | 各末端设备 | 门锁、灯光、空调、音响、传感器等 |

**网络隔离**：三个VLAN——客人网络（VLAN 20，仅互联网）、设备网络（VLAN 10，IoT设备）、办公网络（VLAN 10，收银+店员端）。

- 茶室默认播放背景音乐；客人蓝牙配对后自动切换至蓝牙音源

- 会议室可切换至K歌模式（电视+K歌App+无线麦克风）

### 4.3 VLAN规划

| VLAN ID | 名称 | IP段 | 设备 |
|---------|------|------|------|
| VLAN 10 | 办公/设备 | 192.168.10.0/24 | AP管理口、收银POS、HA主机、NVR、DSP、门锁网关、RS485主机、软路由管理口 |
| VLAN 20 | 客人WiFi | 192.168.20.0/24 | 客人手机上网、投屏 |

防火墙规则：
- VLAN 20 → VLAN 10：禁止访问
- VLAN 10 → VLAN 20：禁止主动发起（仅HA可推送通知）
- 启用AP客户端隔离：禁止同SSID下客人设备互访（防误投屏）

### 4.4 设备选型优先级

| 优先级 | 设备 | 理由 |
|--------|------|------|
| **P0（一期必装）** | 智能门锁、RS485灯光控制、HA主机、Zigbee网关 | 核心服务流程，无替代方案 |
| **P0（一期必装）** | 软路由、PoE交换机、AP | 网络基础设施，所有设备依赖 |
| **P1（一期必装）** | VRF空调网关、温湿度传感器、毫米波雷达 | 提升客人体验，降低能耗 |
| **P1（一期必装）** | DSP矩阵、功放、音箱、蓝牙模块 | 背景音乐为茶室基本体验 |
| **P2（一期选装）** | 电动窗帘 | 提升品质，但可通过手动替代 |
| **P2（一期选装）** | POE摄像头+NVR | 监控安防，可在开业后一周内补装 |
| **P3（二期考虑）** | 无线麦克风、紧急按钮 | 会议室功能扩展，非核心需求 |

## 5. 安装时序与施工注意事项

### 5.1 安装时序

```
第一轮（装修/改造阶段）：
  ① 布线工程
     - 六类网线：机柜→每个AP、摄像头、收银POS、会议室电视、DSP（约150m）
     - RS485线（2芯屏蔽双绞线）：配电箱→各房间继电器模块手拉手（约100m）
     - 音箱线（2芯喇叭线）：功放→每个吸顶音箱（约100m）
     - 窗帘电源线：每个窗位预留零火线
     - 门框预埋：锁体安装孔
  ② 安装预埋件
     - 机柜底座、AP底盒、摄像头支架
     - RS485继电器模块（检修口内）
     - 电动窗帘导轨
  ③ 天花预埋
     - 音箱线、雷达线、蓝牙模块线、空调网关线

第二轮（设备进场阶段）：
  ④ 机柜上架：软路由→交换机→HA主机→NVR→DSP→功放→UPS
  ⑤ 安装末端设备：门锁、AP、摄像头、音箱、传感器
  ⑥ 安装空调网关、RS485主控制器（配电箱）

第三轮（联调阶段）：
  ⑦ HA平台搭建：Home Assistant安装、设备发现、实体配置
  ⑧ 场景配置
     - 欢迎场景（开门→亮灯→开窗帘→播欢迎语）
     - 退房场景（关灯→关空调→关音乐→关窗帘）
     - 节能场景（无人30分钟告警→推送店员）
  ⑨ 离线规则配置（断网后本地执行）
  ⑩ 全流程测试（预约→开门→场景→退房→关设备）
```

### 5.2 关键风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| Zigbee信号覆盖不足 | 门锁/传感器离线 | Zigbee网关吸顶安装于走廊中央；施工前用移动网关实测各房间信号强度 |
| 空调协议不兼容 | 无法远程控制 | 施工前确认空调品牌型号与中弘VRF网关兼容性；备选红外遥控方案 |
| 落地窗电动窗帘安装困难 | 施工成本增加 | 确认窗框上方是否有足够安装空间；窗帘导轨需定制长度 |
| 离线密码时钟漂移 | 客人无法开门 | 选择支持NTP自动校时的门锁（通通锁支持）；HA定期通过网关同步 |
| 客人蓝牙抢占DSP通道 | 音源冲突 | HA监听蓝牙连接状态，仅允许已配对设备控制；客人离场自动断开并复位 |
| RS485总线单点故障 | 整条总线设备失联 | 总线长度控制在100m以内；使用屏蔽双绞线；两端加120Ω终端电阻 |

### 5.3 供电保障

| 设备 | 供电方式 | 断电保障 |
|------|---------|---------|
| 软路由、PoE交换机、HA主机、NVR | UPS直供 | ≥30分钟 |
| Zigbee网关 | 交换机PoE供电（通过UPS） | 随交换机保障 |
| 智能门锁 | 自带电池（4×5号） | 续航≥1年，低电量告警 |
| RS485主控制器 | 配电箱取电（通过UPS） | 随UPS保障 |
| 吸顶AP、墙面AP | 交换机PoE供电 | 随交换机保障 |
| POE摄像头 | 交换机PoE供电 | 随交换机保障 |
| DSP矩阵、功放 | 机柜电源（市电） | 无UPS保障（音频非关键业务） |

### 5.4 HA自动化规则概要

| 触发器 | 条件 | 动作 |
|--------|------|------|
| 门锁解锁（房间） | 订单状态=已支付 | 开灯→开窗帘→播放欢迎语→启动空调 |
| 退房（系统通知） | — | 关灯→关空调→关音乐→关窗帘→闭锁 |
| 雷达无人 | 门锁状态=已关闭 & 持续30分钟 | 推送店员告警 |
| 门锁电量<20% | — | 推送换电池提醒 |
| 门锁2小时无通讯 | — | 推送"疑似离线"告警（重要级别） |
| 预约开始前5分钟 | 房间空闲 | 启动空调预冷/预热 |
| 蓝牙设备连接 | 对应DSP通道空闲 | 切换DSP音源至蓝牙通道 |
| 蓝牙设备断开 | — | 恢复背景音乐 |
