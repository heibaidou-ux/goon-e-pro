## 五、验证的全链路闭环

家庭测试最终需要端到端验证的流程：

```
模拟客人预约（手动触发）
  → 生成门禁密码
  → 输入密码开门
  → 门锁上报DoorOpen事件到HA
  → HA触发Welcome场景（开灯→调温→播音乐）
  → 雷达检测人体存在
  → 模拟退房（手动触发）
  → HA触发EnergySave场景（关灯→关空调→关音乐）
  → 门锁闭锁
```

**通过标准**：上述全链路在无人工干预下，HA自动完成所有设备控制动作，从开门到Welcome场景执行完毕 ≤ 3秒。

## 六、关键风险与应对

| 风险 | 应对 |
|------|------|
| Zigbee信号受家庭WiFi干扰 | 测试时选择干扰最小的Zigbee频道（11/15/20/25） |
| RS485从设备地址冲突 | 每个设备上电前先单独配置唯一Modbus从站地址 |
| 门锁离线密码算法不兼容 | 确认通通锁具体型号的离线密码机制；备选Yeelight智能门锁 |
| M720q噪音影响家庭环境 | 放置于阳台/储物间等隔离空间；风扇转速可BIOS调低 |

## 七、采购决策待确认

1. 音箱和蓝牙模块是否纳入家庭测试？（P6音视频阶段可推迟到现场调试）
2. 门锁具体型号：通通锁哪款？需确认离线动态密码算法细节
3. 软路由是否有替代品？（如树莓派4B刷OpenWRT）——NanoPi R5S优势是双2.5G网口和金属外壳散热
4. 采购渠道：电商 vs 供应商直采？建议核心设备（M720q、R5S）走京东自营，IoT设备走淘宝/1688
