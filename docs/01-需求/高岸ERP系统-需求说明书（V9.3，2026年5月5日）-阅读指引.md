# 高岸ERP系统-需求说明书·阅读指引

**关联文档**：《高岸ERP系统-需求说明书（V9.3，2026年5月5日）》  
**本文档作用**：帮助读者快速理解需求说明书的结构，按角色直达相关章节，无需通读全文。

---

## 一、文档结构一览

| 章节 | 内容 | 页数占比 |
|------|------|---------|
| **第一章 引言** | 项目背景、核心目标（G1全自动月结、G2物联网自动化）、业务术语、业务域关系图 | ~5% |
| **第二章 功能结构** | 功能树总图（脑图+目录树），14个业务域的完整功能清单 | ~10% |
| **第三章 核心业务流程** | **8个泳道图**：营销→预约消费→月结→审批→考勤→巡检→采购→故障处理 | ~20% |
| **第四章 详细功能需求** | **14个业务域**逐域展开：营销、消费、收入、支出、会员、对账、巡检、资产、人事、采购库存、报表、多门店、物联网、预警AI | ~50% |
| **第五章 账务结算自动化** | 结算周期、资金流向、11个收款渠道+8个支出科目规则、月结报告 | ~10% |
| **第六章 非功能性需求** | 性能/可靠/安全/扩展/维护/易用/工作流引擎、**物联网实现路径**（6.8节） | ~5% |

## 三、核心要点速览（100字）

> 高岸ERP围绕**两大目标**构建：G1全自动月结（收入自动归集→对账→生成报告）、G2物联网自动化（预约→开门→场景→退房全链路联动）。覆盖**14个业务域**，原生支持**多门店连锁**架构，按**标准会计科目**组织收支。当前状态：正式评审版（V9.3）。

---

## 四、文档使用建议

1. **首次阅读**：按角色路径读标注★★★的章节，约30-60分钟可掌握核心
2. **快速查找**：通过第二章功能树定位关心的业务域，再跳转第四章对应小节
3. **深究财务**：第五章结算规则是财务自动化的核心，建议财务人员逐条确认
4. **配套文档**：阅读时可对照《系统技术架构设计（ARC-01）》看实现方案、《物联网设备清单（IOT-01）》看设备落地
